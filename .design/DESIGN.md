## Adopted

- Paper surface, ink text, accent colour, border, spacing, radius, typography, and component role values from `vendor/ontwerp/values/`.
- `material.surface.paper-grain` for page, panel, empty-state, and card surfaces.
- `component.button.ink-press` for app-owned buttons and icon controls.
- Field-as-gutter treatment for filter controls where the layout allows it.
- `motion.clock.stepped` for peek-card, filter-sheet, marker, and app-owned control motion.
- Mono utility marks remain uppercase with tracking; user-facing prose remains lowercase.
- Nature-as-behaviour principle for deterministic map rosettes around collocated company pins.

## Deviations (by exception only)

### Adapted

- Mapbox map canvas is repainted at runtime into the paper world via dedicated muted map tones (`--map-land/-warm/-water/-road/-label`) — the wine tint held to roughly a fifth of the page-surface saturation so the basemap reads as calm paper, not a red glow, while the page surfaces stay fully wine. App-owned overlays, controls, markers, and states carry ontwerp values on top.
- Score badges and pentagon scores, filter histograms, and geolocation markers are app-specific map UI; markers are tactile paper tags (paper tooth, ink score, accent indicator dot, thick bottom border, Ben-Day dot screen, stepped press), selected pins invert to ink.
- A soft warm bloom (`atmosphere.weather` blooms) plus paper grain is applied only at the map corners, kept transparent in the centre so it never competes with marker legibility; with the basemap now muted, the corner bloom carries the full wine warmth (accent + deep-surface stops) as the only saturated colour on the map field; the breathing-grid is still omitted over the map.
- The peek card and filter panel header share the same softened light-wine overlay surface (`--filter-surface`) so the bottom overlays read as one family; the peek card identity tile uses a faint accent wash (`--wash-wine`) so it reads as themed rather than the cream baseline. The filter sheet inset filter cards stay on a quieter surface (`--filter-card-surface`) so the controls — not the panel — carry the colour.

### Omitted

- `motion.weather.wind`, `motion.weather.rain`, and related weather particle recipes: no product use case in the map overview yet.
- `state.done.ripe`: no task-completion workflow yet (the ikigai run ends on a results view, not a "done" indicator). `state.loading.germinating` is now adopted by the ikigai matching flow (see Extended `.ikigai-fullscreen`).
- Theme swapping recipes: the app ships one pinned paper/ink theme.

### Extended

- `.map-rosette-pins`: app-specific collocated marker layout; follows nature-as-behaviour and deterministic phyllotaxis.
- `.score-badge` / map cluster markers: app-specific compact map marks; follows component badge and ink-press principles.
- `.score-pentagon`: app-specific five-axis data visualisation; follows semantic values and distinct no-signal state handling.
- `.filter-sheet`: app-specific mobile bottom sheet; follows paper surface, field/gutter, badge, and stepped-motion recipes; shares the peek card's exact stepped slide in/out plus a fading backdrop so the two sheets feel like one object.
- `material.pigment.multiply-blot` on cluster markers: a cluster is rendered as pure pigment — no paper tile, border, or button chrome. A soft organic blot of layered wine/ink dye (accent-soft + ink-soft + accent-base) in a near-circle bleed carries the count in white, following the zoo pigment plate.
- `.filter-chip` active state: a selected tag fills with the accent colour, inverts to on-ink text, and holds the pressed-down state (raised bottom border collapsed), so "selected" is unmistakable; taps also snap down immediately via a transition-free `:active` press.
- `.map-atmosphere`: corner-only warm bloom + paper grain over the recoloured basemap; carries the wine warmth that the basemap itself no longer does.
- `.map-reveal-surface` first-load transition: the releasing paper cover and the bloom that shows through carry wine + amber pigment (accent-soft, amber, accent-base) so the page-load reveal is themed, not a plain white wipe.
- `.favorite-toggle`: app-specific saved-state treatment for icon buttons; reuses the active filter-chip accent fill and held pressed state so saved companies read as selected without introducing a new colour or glyph system.
- `FavoritesPage`: app-specific local shortlist page; uses the warm grain page surface with `ontwerp-card` list items on the wine ramp's `--color-surface-warm` token (`.is-warm-surface`), mono count marks, compact icon buttons, a peek-card-style identity header (favicon tile + name over locality, save star top-right), and the shared focus meter as the per-axis compass summary; the whole card is a stretched link to the detail page (like the peek card), with the save star held above it.
- `.focus-meter`: app-specific three-bar focus-level indicator (filled bars = level, hollow + dashed = no signal); reuses ink/border tones only, the shared focus-level projection that the pentagon rings already define. Used on the detail rows, favorites cards, and the filter sheet's selected minimum.
- `.filter-levels` strip: app-specific four-column axis level control (none/weinig/gemiddeld/veel); the columns are the distribution bars and the minimum selector at once, aligned 1:1 so the at-least boundary lands on a bar edge. Included columns carry a continuous accent rule; follows the badge/field and ink screen recipes, no new tokens.
- `.axis-info-dot`: a tiny square per-axis "?" link sized as a mono utility mark (not a button, no radius — sharp corners per the system), placed beside the axis name so it reads as "explain this axis".
- `.favorites-switch`: a squared paper toggle switch (accent fill + stepped sliding knob) for the favorites-only filter at the top of the sheet; follows ink-press + stepped-motion, no rounded-pill deviation.
- `.pin-favorite-mark`: a small accent star pinned to a saved company's map score tag; reuses the accent fill and paper ring of the active-filter count badge.
- `.byok-sheet`: app-specific bring-your-own-key setup sheet; reuses the filter sheet's paper surface, stepped entry, ink-press buttons, boxed badge, and field-as-gutter pattern for provider/model/key/allowance controls. The Ikigai map chrome entry reuses `ontwerp-icon-button` and the existing left-chrome grouping, with no new token values. Select fields (`.byok-select`) drop the native OS chevron (`appearance: none`) and carry a line-drawn ink caret in the app's own glyph idiom (the `AxisGlyph`/`.axis-chevron` family) instead of a framework default (anti-goal 1); field focus echoes the ontwerp `.field:focus-within` recipe by taking the gutter label and caret to the accent. A reached local-allowance ceiling marks its own field (`.byok-field.is-reached`) with the accent rule + accent label and surfaces a dashed-accent actionable note (`.byok-allowance-note`, reusing the `.byok-status.is-error` dashed idiom) co-located with the amount input — so when an `allowance_exceeded` retry re-gates the visitor into the sheet, the fix is where the eye lands. No new raw token values.
- `.ikigai-sheet`: app-specific matching flow sheet; reuses `ontwerp-sheet`, field-as-gutter text areas, boxed badges, tactile buttons, warm paper result cards, and stepped sheet entry. Axis preference controls reuse the existing focus-level vocabulary and accent selected state; result cards derive current company fields rather than storing snapshots.
- `.ikigai-menu-sheet` / `.ikigai-fullscreen`: the matching flow reworked as a two-surface premium experience. The run-selection menu stays a bottom sheet (`ontwerp-sheet` + the peek/filter-sheet drag handle and stepped entry) listing prior runs and a resume card; once a run starts it becomes a full-screen stepped wizard on the same warm grain surface as the detail page. The wizard reuses ink-press buttons, boxed mono marks, the focus-level axis controls, and warm result cards, and adds: a small seed/phyllotaxis progress rail of circles (one per question step, accent = current, ink = done); large `heading-lg` question prompts with an accent mono eyebrow naming the ikigai circle; quiet "+example" chips that append to the answer; and the ontwerp `state.loading.germinating` seed head (ported from `vendor/ontwerp` phyllotaxis, fill `color.text.soft`, Vogel golden-angle stagger) shown while the model derives work fields and judges matches, with a reduced-motion rest state. A failed pass (bad key, allowance reached, model/network error) no longer swaps in a separate boxed error card: the germinating surface itself becomes the error state (`.ikigai-loading.is-error`) — the seed head halts and settles to a quiet tone (growth stalled = something is wrong, per nature-as-behaviour), and the same centred surface carries the message (`#ikigai-error`) plus the fix/back actions (`.ikigai-loading-actions`); for key/allowance failures the fix action re-gates through the BYOK sheet. No raw colour, radius, spacing, type, or motion values were introduced.
